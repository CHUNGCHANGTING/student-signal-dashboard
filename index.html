收
Iron Condor 詳細規則
IC_INDEX（SPX/QQQ）與 IC_EQUITY（個股）雙路由

路由判斷：n8n pickModule() 依 is_index 欄位分流。is_index: true → IC_INDEX；is_index: false → IC_EQUITY。兩條路由共用相同進場 IV/DTE 門檻，但個股 wing width 與 BP usage 有獨立限制。

進場條件
IVR ≥ 30
VIX 或個股 IV Rank ≥ 30；市場處於高波動定價期
DTE 21–45
Theta 衰減加速區間；避免太短 (gamma risk) 或太長 (vega risk)
Regime = Neutral
SPX 在 SMA20/50 附近震盪；無明顯趨勢訊號
無重大事件
未來 5 日無 FOMC、CPI、財報（個股 IC 必查）
IC 進場確認0 / 4
出場規則
獲利
達到 50% 最大獲利時平倉
→ 主動鎖利，不貪心等到期
時間
距到期 ≤ 7 DTE 強制平倉
→ 避免 gamma 風險爆炸
止損
虧損達 2× credit 時停損
→ 硬性規定，不得翻倉或 roll 來逃避

進場條件

足夠 OTM 但不能太遠導致 credit 太少
Credit ≥ 30% width
例：5 pts spread → 至少 $1.50 credit

進場條件

IVR ≥ 27
高 IV 環境 call premium 豐厚

OTM call；不追頂
Credit ≥ 30% width
最低收租門檻

攻
Bull Call Debit Spread 詳細規則
偏多突破進攻 — 趨勢確認後使用
進場條件
Regime = Bull（強勢）
價格突破關鍵阻力，成交量配合
IV 低到中（IVR < 50）
買方策略需要 IV 不能太高；否則 premium 太貴
Long call delta 0.50–0.60
ATM 附近，delta 充足但不過度暴露
Debit ≤ 40% spread width
控制買入成本；確保 R:R 合理
BCDS 進場確認0 / 4
攻
Bear Put Debit Spread 詳細規則
偏空跌破進攻 — 趨勢確認後使用
進場條件
Regime = Bear（強勢）
價格跌破關鍵支撐，成交量放大
IV 低到中（IVR < 50）
同 BCDS；IV 太高時不用買方策略
Long put delta -0.50 — -0.60
ATM 附近的 put；避開深度 ITM
Debit ≤ 40% spread width
控制買入成本

攻
Credit Vertical（次要）詳細規則
IV 極高 + 方向明確時才用；非主力策略

注意：Credit Vertical 在極端行情時使用。一般情況下優先用 IC、BPCS 或 BCCS。只有 IVR ≥ 60 且方向極明確時才考慮。

進場條件
IVR ≥ 60
極高 IV 環境；premium 豐厚到值得單邊方向押注
方向極明確
技術面 + 基本面都支持同一方向；不能只有其中一個
個股標的（非指數）
指數環境用 IC 或單邊 spread 更有效率
Credit ≥ 30% width
CV 同樣需要達到最低收租門檻
